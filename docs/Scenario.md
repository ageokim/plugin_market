> 각 시나리오는 "사용자가 하는 행동 → 화면에 보이는 것" 기준으로 그렸다. 화면 목업: [mockup/](mockup/)

## 0. 등장 요소 — 사용자가 만나는 것들

```mermaid
flowchart LR
    U["👤 사용자"] -->|"브라우저로 사용"| WEB["🌐 웹 화면 (localhost:8765)<br>사이드바 + claude 챗 + 내장 터미널"]
    WEB -->|"API·WebSocket"| FL["🖥️ Flask 서버<br>(run.sh가 내 PC에서 실행)"]
    FL -->|"스캔·설치"| GH["☁️ GitHub (단일 서버, 여러 org)"]
    FL -->|"등록"| CL["🤖 local claude"]
```

사용자가 직접 만지는 것은 **두 가지뿐**이다:
1. 터미널에서 `run.sh` 실행 (최초 또는 서버가 꺼져 있을 때)
2. 브라우저 화면 (나머지 전부 — **창을 닫으면 서버도 자동 종료**)

## 시나리오 1 — 처음 시작하기 (최초 실행)

```mermaid
sequenceDiagram
    actor U as 👤 사용자
    participant T as 터미널
    participant B as 브라우저
    participant FL as Flask 서버

    U->>T: ① git clone + ./run.sh
    Note over T: 자동 셋업 (멱등)<br>python 확인 → 라이브러리 확인<br>(이미 있으면 pip 생략)<br>pm 명령 PATH 등록
    T->>FL: ② Flask 백그라운드 기동 (localhost:8765)
    T->>B: ③ 브라우저 자동 오픈
    B-->>U: ④ 로그인 창 표시 (시나리오 3)
    Note over B,FL: 이후 페이지가 2초마다 생존 신호(heartbeat)를<br>보내고, 창을 닫으면 서버가 스스로 꺼진다 (시나리오 6)
```

**사용자가 하는 일:**

| 단계 | 행동 | 화면/결과 |
|---|---|---|
| ① | `git clone … && ./run.sh` | 터미널에 셋업 진행 로그 |
| ② | (없음 — 자동) | 브라우저가 저절로 열림 |
| ③ | (없음 — 자동) | 로그인 창 |

> 환경이 미비하면 로그인 창 대신 **체크리스트 화면**이 먼저 뜬다 → 시나리오 2.

## 시나리오 2 — 환경이 미비할 때 (체크리스트)

```
┌──────────────────────────────────────────────┐
│  환경 셋업 체크리스트                        │
├──────────────────────────────────────────────┤
│  ✅ python 3.12 발견                         │
│  ✅ 라이브러리 (flask·requests·SDK)          │
│  ❌ claude CLI 없음                          │
│      원인: claude 명령을 찾을 수 없습니다    │
│      👉 npm install -g @anthropic-ai/claude-code  [복사] │
│  ❌ pm PATH 미등록                           │
│      👉 ./env/setup_linux.sh                 [복사] │
│                                              │
│               [ 🔄 재검사 ]                  │
└──────────────────────────────────────────────┘
```

```mermaid
flowchart LR
    A["체크리스트 화면<br>✅/❌ 항목별 표시"] --> B["❌ 항목의<br>추천 명령어 복사 버튼 클릭"]
    B --> C["터미널에 붙여넣어 실행"]
    C --> D["🔄 재검사 버튼"]
    D -->|"아직 ❌ 남음"| A
    D -->|"모두 ✅"| E["로그인 창으로 자동 진행"]
```

**사용자가 하는 일:** ❌ 항목 옆의 명령을 복사 → 터미널에서 실행 → 재검사. 그게 전부다.

> 단, python·라이브러리·포트처럼 **웹 화면 자체를 띄우는 데 필요한 항목**이 문제라면
> 브라우저가 아니라 `run.sh`의 터미널 로그가 같은 안내를 한다 — 조치 후 `run.sh` 재실행.

## 시나리오 3 — 로그인 (ID / PWD 두 칸)

```
┌──────────────────────────────────────┐
│      🧩 Plugin Cafe — 로그인       │
├──────────────────────────────────────┤
│  ID    [ ageokim                  ]  │
│  PWD   [ ●●●●●●●●●●  (PAT)       ]  │
│                                      │
│            [ 입장하기 ]              │
│                                      │
│  · 입장하면 자동 저장되어 다음부터  │
│    이 창 없이 바로 시작됩니다        │
└──────────────────────────────────────┘
```

- **딱 두 칸**: ID + PWD(PAT — GitHub이 암호 인증을 폐지해 비밀번호 대신 토큰). org 주소는 여기가 아니라 **챗 입력줄의 ☕ 팝오버에서** 입력한다(시나리오 3.5) — 여러 org를 자유롭게 추가·제거하기 위함.
- 로그인 성공 시 `data/credentials.json`에 **자동 저장** → 다음 실행부터 로그인 창 생략(시나리오 6).
- 검증: 토큰이 유효한지 + 토큰 주인이 입력한 ID와 같은지. (org 권한은 org를 추가하는 순간 검사)
- 최초 실행이라 GitHub 서버 주소를 아직 모르는 경우: 로그인 정보는 보관되고, **첫 org URL을 추가하는 순간** 서버가 정해지면서 한꺼번에 검증된다. 그때까지는 org 추가 외의 기능이 잠긴 "미검증" 상태이며, **검증이 끝나야 자동 저장도 이뤄진다**.

## 시나리오 3.5 — organization 추가 (다중 org 혼합 사용)

```
챗 입력줄 (전송 버튼 왼쪽 ☕ 아이콘 클릭 시 위로 팝오버):
        ┌──────────────────────────────────────┐
        │ ☕ ORGANIZATION 추가 — 원두를 담아요  │
        │ [ https://github.xxx.xxx/org-b   ][+] │
        └──────────────────────────────────────┘
[☕] [ claude에게 메시지…                ] [▶]
 ↑ 바깥 클릭/Esc 하면 아이콘으로 접힘

사이드바 (등록된 org 레시피 카드):
│ ▽ org-a  재료 3가지 RECIPE  🔄 ✕    │
│ ▽ org-b  재료 5가지 RECIPE  🔄 ✕ ←추가│
```

```mermaid
flowchart TD
    A["챗 입력줄 ☕ 아이콘 → 팝오버에<br>org URL 붙여넣고 + 클릭"] --> B{"권한 검사<br>(내 계정이 이 org 멤버인가?)"}
    B -->|"통과"| C["org 등록 + 자동 스캔<br>→ #plugin #release repo들이 그룹으로 표시"]
    B -->|"권한 없음"| D["❌ '이 organization에 권한이 없습니다'<br>— 등록되지 않음, 목록에 안 보임"]
    B -->|"다른 서버 주소"| E["❌ '등록된 서버(github.xxx.xxx)와 다릅니다'<br>(단일 서버 정책)"]
    C --> F["여러 org의 플러그인을<br>한 목록에서 혼합 설치·사용"]
```

- URL은 저장되어 다음 실행에도 유지된다. org마다 🔄(재스캔)·✕(제거) 버튼 — **제거는 그 org의 설치된 플러그인까지 함께 삭제**하고 **preset에 담겨 있던 해당 멤버도 자동으로 빠지며**, 삭제될 개수를 보여주는 확인을 거친다.
- **권한이 있는 org만 보인다** — 등록할 때 검사하고, 매 시작 때 재검사해서 권한을 잃으면 잠금 표시.

## 시나리오 4 — 플러그인 검색과 설치 (사이드바)

```mermaid
sequenceDiagram
    actor U as 👤 사용자
    participant SB as 사이드바
    participant GH as ☁️ GitHub
    participant CL as 🤖 local claude

    U->>SB: ① org 그룹의 🔄 스캔 (또는 org 추가 직후 자동)
    SB->>GH: repo 목록 조회 (org별)
    GH-->>SB: #plugin #release 붙은 repo들
    SB-->>U: ② org별 그룹으로 목록 표시

    SB->>CL: 링크 생성 (.claude/plugin_roots 등)<br>plugin.json 있으면 등록도 병행
```

- 어느 org의 플러그인이든 설치하면 **같은 방식으로 claude에 등록**된다 (혼합 사용).

## 시나리오 5 — claude 사용 (주 화면: 챗 + 내장 터미널)

주 화면 상단의 **탭으로 뷰를 전환**한다 ([새 대화] 버튼 옆 — Workflow 탭은 시나리오 9):

```
[ ✦ Claude | ❯ 터미널 | ⌁ Workflow ]  [+ 새 대화]  ← 탭 클릭으로 전환
                          (터미널 탭에선 버튼이 [+ 새 터미널]로 바뀜)
┌───────────────────────────────────┐
│ (Claude 탭)  SDK 기반 대화          │
│  > 코드리뷰 해줘                   │
│  ⏺ plugin-a: code-review skill …  │
└───────────────────────────────────┘
┌───────────────────────────────────┐
│ (터미널 탭)  진짜 셸 — 브라우저 안  │
│  $ pm list                        │
│  $ claude        ← 완전한 대화형   │
└───────────────────────────────────┘
```

- **Claude 탭**: 브라우저에서 claude와 바로 대화. 설치된 플러그인의 skill이 자동 적용. 액션 버튼 = [새 대화].
- **터미널 탭**: 진짜 셸이다 — `pm` 명령, `claude`(완전한 대화형) 모두 그대로 실행. 원격(SSH)에서도 브라우저만 있으면 동일하게 동작. 액션 버튼 = [새 터미널].

### ⚠️ 적용 시점 규칙

- **standalone plugin(사내 기본)**: 링크가 생기는 즉시 경로 참조 가능 — 재시작 불필요.
- **native plugin(plugin.json 보유)**: skill/command는 claude가 **시작할 때** 읽어들인다:

| 상황 | 새 플러그인 동작? |
|---|---|
| Install/Enable **후에 새로 시작한** 세션 | ✅ 바로 동작 |
| Install/Enable **전부터 열려 있던** 세션 | ❌ 재시작 필요 |

### 🔄 claude 재시작 방법

- **챗**: [➕ 새 대화] 버튼 클릭 — 그게 전부 (Install 직후 "새 대화부터 적용됩니다" 안내 표시)
- **내장 터미널**:
  ```
  > exit          ← claude 종료, 셸로 복귀
  $ claude        ← 다시 실행 = 새 세션 (플러그인 로딩)
  ```
  셸에서 한 번 더 `exit`하면 그 터미널 세션만 끝나고 "[새 터미널]" 버튼이 뜬다 — 페이지·서버에는 영향 없음.

**재시작이 필요 없는 것들:** Flask 서버 ❌ · 브라우저 페이지 ❌ · run.sh 재실행 ❌ — claude 세션만 새로 시작하면 된다.

## 시나리오 6 — 다시 실행 / 끝내기

```mermaid
sequenceDiagram
    actor U as 👤 사용자
    participant T as 터미널
    participant B as 브라우저
    participant FL as Flask

    U->>T: ./run.sh
    Note over T: 셋업 전부 "이미 충족 → 건너뜀"<br>(pip 안 돌음, 몇 초 내 기동)
    T->>B: 브라우저 자동 오픈
    Note over B: credentials.json으로 자동 로그인<br>+ 등록된 org들 권한 재확인
    B-->>U: 로그인 창 없이 바로 메인 화면 ✅
    U->>B: (다 쓰고) 브라우저 창 닫기
    B--)FL: 닫힘 신호 + 생존 신호 중단
    Note over FL: 10초 유예 후 서버 자동 종료<br>(닫힘 신호 유실 시 최대 ~20초 · 터미널 정리 불필요)
```

- **켜기**: `./run.sh` 한 번 — 자동 로그인(저장된 credentials.json)으로 바로 메인.
- **끄기**: 그냥 **브라우저 창을 닫으면 끝** — 서버가 스스로 감지하고 종료된다. 새로고침은 10초 유예 덕에 끊기지 않는다.
- **토큰이 만료·회수됐으면**: 자동 로그인이 실패하고 로그인 창(값은 채워진 상태)으로 돌아온다.
- **특정 org의 권한만 잃었으면**: 로그인은 유지되고 그 org만 잠금 표시된다 — 어느 쪽이든 확인은 매번 수행되므로 우회는 없다.

## 시나리오 7 — 플러그인 관리 (사이드바 버튼 기준 상태 변화)

```mermaid
stateDiagram-v2
    Available: ⚪ 재료 없음 (org에 존재)
    Installed: 🟡 보관중 (내 PC에 있음)
    Enabled: 🟢 추출중 (claude가 사용)

    Available --> Enabled : 담기 클릭
    Enabled --> Installed : 끄기 클릭
    Installed --> Enabled : 켜기 클릭
    Installed --> Available : 삭제 (확인 후)
    Enabled --> Available : 삭제 (확인 후)
```

| 버튼 (내부명) | 일어나는 일 | 이후 claude에서 |
|---|---|---|
| 담기 (install) | 다운로드 + 링크 생성(켜기) | standalone 즉시 · native skill은 새 세션부터 |
| 끄기 (disable) | 링크 제거 (파일은 유지) | standalone 즉시 · native는 새 세션부터 |
| 켜기 (enable) | 링크 재생성으로 다시 켜기 | 새 세션부터 (plugin의 경로 참조는 즉시 §6.2) |
| 🗑 삭제 (uninstall) | **"삭제할까요?" 인라인 확인 후** 끄기 + 등록 해제 + 파일 삭제 → 행이 '재료 없음'으로 복귀 | 안 보임 |
| 업데이트 (update) | 새 버전 받아 재등록 (켜짐/꺼짐 상태는 그대로) | 새 세션부터 새 버전 |

> 재료 없음/보관중/추출중은 화면 표시명(카페 카피)이고, 내부 상태명(Available/Installed/Enabled)과
> CLI 출력은 영문이다 (Architecture §12.2).

- **Inspect**를 열면 각 플러그인의 실제 상태(파일 존재·등록·켜짐, 버전 차이)를 표로 확인 — 뭔가 이상할 때 여기부터 본다.

## 시나리오 8 — Preset: 나만의 플러그인 세트

자주 함께 쓰는 플러그인들을 **preset**으로 묶어두면, 하나하나 누르지 않고 세트째 다룰 수 있다.

```
사이드바 Preset 섹션:
┌─────────────────────────────────────────┐
│ ☰ TODAY'S PRESET — 나만의 메뉴 [+ 새 preset]│
│  ● 코드리뷰 세트 (3)      [전환] ⋯      │  ← ● 전부 켜짐
│  ○ 문서작업 세트 (2)      [전환] ⋯      │  ← ○ 꺼져 있음
└─────────────────────────────────────────┘
   ⋯ 메뉴: 일괄 설치 · 켜기 · 끄기 · 삭제 / 멤버 편집 / 정의 삭제
```

**만들기와 멤버 추가:**

```mermaid
flowchart LR
    A["[+ 새 preset] 클릭<br>이름 입력: '코드리뷰 세트'"] --> B["preset 행 ⋯ 메뉴 → 멤버 편집<br>카탈로그 전체에서 ✓ 토글로 담기"]
    A --> B2["또는: 플러그인 행에 마우스 올리면<br>나타나는 '+ preset' 아이콘"]
    B --> C["멤버 구성 완료<br>(org 상관없이 섞어 담기 가능)"]
    B2 --> C
```

**세트째 실행 — 버튼 하나로 멤버 전체에 동시 수행:**

| 동작 | 결과 |
|---|---|
| 일괄 켜기 | 멤버 전부 켜짐 — **재료 없음(미설치) 멤버는 자동으로 담기까지** 하고 켠다 |
| 일괄 끄기 | 멤버 전부 꺼짐 (파일 유지) |
| 일괄 삭제 | 인라인 확인 후 멤버 전부 삭제 → 각 행이 '재료 없음'으로 |
| **[전환]** | **멤버만 켜고, 멤버가 아닌 설치본은 전부 끔** — "코드리뷰 세트" ↔ "문서작업 세트"를 작업 모드처럼 갈아탐 (삭제는 안 함) |

```mermaid
flowchart LR
    A["'문서작업 세트' [전환] 클릭"] --> B["멤버 2개: 켜짐<br>(미설치였다면 설치부터)"]
    A --> C["멤버 아닌 설치본: 전부 꺼짐"]
    B --> D["'새 대화부터 적용됩니다' 안내<br>→ ➕ 새 대화에서 그 세트로 작업"]
    C --> D
```

- 일부 멤버가 실패해도(네트워크 등) 나머지는 계속 진행되고, **멤버별 성공/실패 요약**이 표시된다.
- preset의 상태 점은 멤버들의 실제 상태에서 계산된다 — ● 전부 켜짐 / ◐ 일부 / ○ 꺼짐.
- "정의 삭제"는 묶음 목록만 없애고 플러그인은 건드리지 않는다 (멤버까지 지우려면 일괄 삭제 먼저).
- CLI로도 동일: `pm preset enable 코드리뷰세트`, `pm preset apply 문서작업세트` 등 (Architecture §7).

## 시나리오 9 — Workflow 탭: claude가 일하는 걸 지켜보기

플러그인이 실행하는 워크플로우(도구 호출·서브에이전트)가 **지금 어디까지 진행됐는지**
그래픽 타임라인으로 본다. 챗에서 시킨 것이든, 내장 터미널에서 직접 돌린 claude든 다 잡힌다.

```
[ ✦ Claude | ❯ 터미널 | ⌁ Workflow ]  [기록 지우기]
┌────────────────────────────────────────────┐
│ ⌁ 세션 a3f2 (챗) · 진행 중                  │
│ │                                          │
│ ● 프롬프트: "코드리뷰 해줘"        10:02:01 │
│ ● Read src/app.py ✓ (0.2s)        10:02:03 │
│ ◐ Bash pytest -q  ◉ 진행 중… (펄스)10:02:05 │
│ │  ┌─ 서브에이전트: code-reviewer          │
│ │  │  ● Grep "TODO" ✓                      │
│ │  └─ ○ 대기…                              │
│ ● mcp[plugin-a] review ✓  ← 플러그인 배지  │
└────────────────────────────────────────────┘
   필터: [전체 | 플러그인만]   자동 스크롤 ⤓
```

```mermaid
sequenceDiagram
    actor U as 👤 사용자
    participant C as claude (챗 또는 터미널)
    participant FL as Flask 서버
    participant W as Workflow 탭

    U->>C: "코드리뷰 해줘" (플러그인 skill 동작)
    C--)FL: hook: 도구 실행 시작 (자동, 비차단)
    FL--)W: SSE 푸시
    W-->>U: ◉ 진행 중 단계가 펄스로 표시
    C--)FL: hook: 도구 완료
    FL--)W: SSE 푸시
    W-->>U: ✓ 완료로 전환 + 경과시간
```

- **행동은 탭 클릭뿐** — 수집은 자동이다(설치·설정 불필요, 프로젝트에 내장).
- 진행 중 단계는 **펄스**, 완료 ✓, 실패 ✗. 서브에이전트 단계는 **들여쓰기**로 구분.
- 플러그인이 제공한 도구는 **플러그인 배지**가 붙는다 — [플러그인만] 필터로 그것만 골라 볼 수 있다.
- 기록은 서버 메모리에만 있다(최근 세션 일부) — 서버를 끄면 사라지고, [기록 지우기]로 즉시 비울 수 있다.
- 터미널에서 대화형 `claude`를 처음 실행할 때 "프로젝트 hook을 신뢰하시겠습니까?" 확인이 1회 뜰 수 있다 — 이 수집 장치(관찰 전용)에 대한 승인이다.
